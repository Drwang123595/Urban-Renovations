# 自动化标签管理与纯净输出计划 (修正版)

根据您的要求，输出文件（output）**不能包含原始标签**，且 `labels` 文件夹自动存放原始带标签文件以供评估。

## 1. 核心逻辑变更

### 1.1 自动归档标签 (Auto-Archive Labels)
*   **时机**：在 `DataProcessor.run_batch` 开始处理之前。
*   **操作**：将选定的输入文件（来自 `Data/train/`）完整复制到 `Data/{task_name}/labels/` 目录下。
*   **目的**：确保评估时有标准答案（Ground Truth），且无需人工手动复制。

### 1.2 纯净输出 (Clean Output)
*   **读取输入**：读取 Excel 为 DataFrame。
*   **过滤列**：在开始处理前或保存结果时，**显式剔除**所有已知的标签列（如 `是否属于城市更新研究`, `Label_xxx` 等），或者仅保留 `Article Title`, `Abstract` 及其他元数据（如 `Year`, `DOI`），加上模型新生成的预测列。
*   **避免污染**：确保生成的 `output/xxx.xlsx` 中**不含**人工标注的正确答案，只包含模型预测结果。

## 2. 修改代码 `src/data_processor.py`

*   **引入 `shutil`**：用于文件复制。
*   **修改 `run_batch`**：
    1.  **复制源文件**：`shutil.copy2(input_path, labels_dir / input_path.name)`。
    2.  **清洗 DataFrame**：
        *   定义一个“保留列白名单”或者“剔除列黑名单”。
        *   或者简单粗暴：只保留 `Article Title` 和 `Abstract`（以及必要的索引列）。
        *   *策略*：在内存中处理时，先从原始 `row` 中删除所有可能的标签列，然后再合并 `extracted` 结果。
    3.  **保存结果**：写入 Excel。
