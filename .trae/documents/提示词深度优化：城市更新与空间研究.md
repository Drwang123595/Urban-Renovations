# 提示词优化方案：城市更新与空间研究精准化

根据您的需求，我们将进一步深入优化提示词，重点解决“城市更新研究”和“空间研究”这两个核心指标的判定准确性。目前的提示词已经有了基础定义，但还可以通过**更具体的排除规则**、**更丰富的场景描述**和**逻辑强化**来提升模型表现。

## 1. 优化目标
1.  **城市更新研究 (Urban Renewal)**: 减少将“一般性城市发展”、“新建项目”或“纯理论研究”误判为城市更新的情况。
2.  **空间研究 (Spatial Study)**: 减少将“仅作为背景提及地名”的研究误判为空间研究，同时确保识别出隐含的空间尺度。

## 2. 详细优化策略

### 2.1 城市更新研究 (Urban Renewal) - 强化“存量”与“干预”
*   **核心逻辑**: 强调 **EXISTING built-up areas** (既有建成区) + **INTERVENTION** (干预/行动)。
*   **增加排除项 (Negative Rules)**:
    *   **New Town Development**: 明确排除新城建设、新区扩张。
    *   **General Urban Growth**: 排除一般性的城市化进程讨论（如人口增长、土地扩张），除非涉及旧区改造。
    *   **Pure Housing Market**: 排除纯粹的房地产价格分析，除非明确提到旧改对房价的影响。
*   **增加正向场景 (Positive Scenarios)**:
    *   补充更多具体的更新行为：`gentrification` (绅士化，常伴随更新), `informal settlement upgrading` (非正规定居点升级), `brownfield remediation` (棕地修复), `adaptive reuse` (适应性再利用).

### 2.2 空间研究 (Spatial Study) - 强化“具体性”与“研究对象”
*   **核心逻辑**: 区分 **Mentioned as Context** (仅作为背景) vs **Conducted in/on** (作为研究对象)。
*   **判定标准**:
    *   如果摘要说 "Using data from Beijing..." -> **Yes**.
    *   如果摘要说 "China is experiencing rapid urbanization..." (泛指) -> **No**.
    *   如果摘要说 "We propose a model and test it in Shenzhen..." -> **Yes**.
*   **增加规则**:
    *   **Data Source Rule**: 只要明确提到了数据来源于特定地点，即视为空间研究。
    *   **Case Study Rule**: 明确的案例研究（Case Study）必然是空间研究。

**Field 2 (Spatial Study) 部分增加：**
```yaml

  - "Contextual Mention" vs "Study Object":
    - If a place is mentioned ONLY to provide general background (e.g., "In rapidly developing countries like China..."), output 0.
    - If the study USES DATA from a place, ANALYZES a place, or DOES A CASE STUDY in a place, output 1.
```
