# Prompt 优化方案：引入专家级 Few-shot

您提供了一个非常详尽、定义严谨且带有高质量正反例的 Few-shot Prompt。我将把这个结构完全融入到现有的提示词体系中，重点替换和增强“城市更新研究方向判定”部分。

## 1. 核心策略：全方位对齐

我将对 `single.yaml`, `cot.yaml`, `stepwise.yaml` 进行以下修改：

### 1.1 `single.yaml` (Single Strategy)
*   **System Prompt**:
    *   **判定口径**: 直接使用您提供的“既有建成区（存量空间）的改造...”定义。
    *   **排除原则**: 显式列出您提供的 3 条排除原则（宽泛议题、纯工程技术、商业背景变量）。
*   **Examples (Few-shot)**:

    *   保持原有的 Tab 分隔输出格式（4个字段）。

### 1.2 `cot.yaml` (CoT Strategy)

    *   要求模型在 `<thinking>` 阶段显式对照您提供的判定口径和排除原则。

### 1.3 `stepwise.yaml` (Stepwise Strategy)

## 2. 实施细节

### 正例映射 (Mapping User Examples to Schema)
您提供的例子只有 Input/Output (1/0)。我需要为这些例子补充后 3 个字段（Spatial Study, Level, Desc）以符合我们的 4 字段输出格式。
*   *Input 1 (Central government role)* -> Output: `1  0  Not mentioned  Not mentioned` (Policy/Governance study, likely no specific spatial analysis)
*   *Input 2 (East Belfast)* -> Output: `1  1  Single Neighborhood/Block  UK-Northern Ireland-Belfast-East Belfast`
*   *Input 3 (Brussels circular)* -> Output: `1  1  Single City  Belgium-Brussels`
*   *Input 4 (Keelung Fire)* -> Output: `1  1  Single City  China-Taiwan-Keelung`
*   *Input 5 (REI consumer)* -> Output: `0  0  Not mentioned  Not mentioned`

这样的映射既保留了您的判定逻辑，又适配了现有的数据结构。
