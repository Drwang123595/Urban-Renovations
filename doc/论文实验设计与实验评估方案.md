## 2. 任务一：城市更新主任务（Urban Renewal Identification）

### 2.1 任务定义

输入：论文题名（`Article Title`）与摘要（`Abstract`）及少量元数据列。  
输出：是否属于城市更新研究（`urban_flag` / `final_label`），并提供解释性字段（证据、规则栈、决策来源、Unknown 机制审计等）。

二分类真值：稳定评估集当前仅闭环 `urban_flag` 相关字段；主题 25 类真值 `theme_gold` 为空时不纳入终局量化闭环。

### 2.2 方法矩阵（research_matrix 三方法并列）

论文主任务对比矩阵在 `research_matrix` 轨道固定包含三类方法：

1. `local_topic_classifier`
2. `three_stage_hybrid --hybrid-llm-assist off`
3. `three_stage_hybrid --hybrid-llm-assist on`

其中 `pure_llm_api` 仅作为附录上界与成本对照，不进入正文稳定对比表。

### 2.3 当前 stable_release 的真实决策链（必须写清楚）

当前 `stable_release` 的 `three_stage_hybrid` 并非“单一分类器”，而是多阶段的精度约束混合判定链。论文必须按真实链路描述，而不是把这些能力写成“未来阶段”：

- `stage1_rule`：高精度规则先行判定（明确正/负样本直接落地）
- `rule_model_fusion`：规则、local、BERTopic 等信号的加权融合与置信度边界控制
- `family gate`：将“是否属于城市更新（U/N/Unknown）”与“族内主题选择”解耦的门控模型（稳定版已参与大量样本的 family 判定与审计）
- `anchor_guard`：对易误报主题增加更新锚点/对象/机制的保护性校验，抑制 FP
- `uncertain_nonurban_guard`：对“疑似非城市更新但证据不足”的样本进行保守处理
- `open_set_recovery`：针对开放集边界样本的受限回收策略（不允许无约束提升为正类）
- `unknown_hint_resolution`：当且仅当多路信号同向且风险可控时，对 Unknown 进行受限回收
- `unknown_review`：残余 Unknown 审核池导出（供人工复核与误差分析）

LLM 在 stable_release 中的角色是“受限提示信号”：

- 仅用于生成 `Unknown` 样本的 family hint
- 不直接覆盖 `topic_final`
- stable_release 的验收合同要求 `llm_used == 0`（只允许 “attempted but not used as final decision”）

### 2.4 指标定义（数学口径）

设稳定评估集大小为 `N`，二分类真值为 `y_i ∈ {0,1}`，预测为 `ŷ_i ∈ {0,1}`。

混淆矩阵：

- `TP = Σ 1[y_i=1 ∧ ŷ_i=1]`
- `TN = Σ 1[y_i=0 ∧ ŷ_i=0]`
- `FP = Σ 1[y_i=0 ∧ ŷ_i=1]`
- `FN = Σ 1[y_i=1 ∧ ŷ_i=0]`

主指标：

- `Accuracy = (TP + TN) / N * 100`
- `Precision = TP / (TP + FP)`
- `Recall = TP / (TP + FN)`
- `F1 = 2 * Precision * Recall / (Precision + Recall)`

Unknown 相关指标（来自 `Eval_Summary.xlsx` 的 `Unknown Rate`）：

- `Predicted Unknown Count`：预测为 Unknown 的样本数
- `Predicted Unknown Rate = Predicted Unknown Count / N`

### 2.5 稳健性与显著性检验（研究矩阵用）

1. Bootstrap 95% CI  
对样本索引进行 `B` 次有放回重采样，分别计算 `Accuracy` 与 `F1` 得到样本分布，取 `2.5%` 与 `97.5%` 分位数作为 95% 置信区间（输出到 `Bootstrap CI`）。

2. McNemar 检验（最优两模型对比）  
对同一真值集上的两模型 `A/B`，令：

- `b = #(A 正确, B 错误)`
- `c = #(A 错误, B 正确)`

采用连续性校正的统计量：

- `χ² = (|b - c| - 1)² / (b + c)`（当 `b + c = 0` 时定义为 0）

并据此给出 p-value（输出到 `McNemar`）。

3. 长上下文顺序敏感性（仅研究矩阵，不进入稳定主结论）  
若采用 `cross_paper_long_context`，必须对三种固定顺序聚合报告：

- `canonical_title_order`
- `shuffle_seed_20260415_a`
- `shuffle_seed_20260415_b`

当 `max_delta_accuracy > 1.5` 或 `max_delta_f1 > 0.015` 时，论文必须标记为 `order_sensitive`。

## 3. 任务二：空间属性抽取（阶段性成果）

空间任务具备工程可运行链路，但当前稳定评估集缺少闭环真值，因此论文仅写“方法体系 + 待标注闭环方案”，不宣称终局量化结论。

- 运行：`scripts/pipeline/main_py313.py --task spatial`
- 输出字段：`Spatial Study` / `Spatial Level` / `Spatial Desc` 等

闭环方案（论文建议）：从 1000 篇中分层抽样 300 篇，双人标注并裁决后形成 `spatial_gold` 子集，使用：

- `Spatial Study`：二分类 `Accuracy/Precision/Recall/F1`
- `Spatial Level`：层级匹配准确率
- `Spatial Desc`：Jaccard >= 0.6 的模糊匹配准确率
