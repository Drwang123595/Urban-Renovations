# 城市更新文献识别论文实验设计与实验评估方案

## 1. 实验目标与研究问题

本文围绕城市更新文献识别系统构建三条并列的实验主线：城市更新识别、空间属性抽取和细粒度主题识别。三项任务共享统一的运行入口、提示词治理机制和离线评估接口，但在当前阶段的闭环程度不同：城市更新识别已经具备稳定评估集和可复现实验结果，空间属性抽取与细粒度主题识别则进入“方法体系已成型、正式量化待补标”的分阶段状态。

结合当前项目设计，论文实验部分拟回答以下四个研究问题。

1. `three_stage_hybrid + LLM assist on` 是否在城市更新主任务上优于 `local_topic_classifier` 和 `three_stage_hybrid + LLM assist off`。
2. `unknown_hint_resolution` 是否能够在不增加误判的前提下降低 Unknown 留存率。
3. 长上下文运行是否存在显著顺序敏感性，从而不适合作为稳定版结论依据。
4. 空间属性抽取与细粒度主题识别是否已经形成可复用的结构化分析框架，并为下一阶段补标闭环提供明确实验接口。

## 2. 统一实验平台与数据口径

### 2.1 运行与评估入口

系统统一采用以下运行链路。

- 运行入口：`scripts/main_py313.py`
- 离线评估：`scripts/evaluate.py`
- 正式汇总工件：`Eval_Summary.xlsx`
- 运行协议约束：`README.md` 与 `doc/evaluation_baseline_20260408.md`

其中，`Eval_Summary.xlsx` 是论文正式引用的唯一量化汇总来源。Accuracy 按 `0-100` 标尺报告，Precision、Recall、F1 保持 `0-1` 标尺；任何 `Accuracy > 100` 的结果均视为无效。

### 2.2 数据集与实验轨道

论文正文仅采用当前仓库中可核实的数据与工件，不引用 `generate_stage_report.py` 中依赖但当前仓库不存在的外部数据集。实验数据划分如下。

| 轨道 | 数据集 | 用途 | 论文口径 |
| --- | --- | --- | --- |
| `stable_release` | `Urban Renovation V2.0_cleaned_article_sample_1000_local_labeled_v2_20260407` | 稳定版验收与主结论 | 正文主证据 |
| `research_matrix` | 同一 1000 篇评估集上的方法矩阵运行 | 方法对比、消融与长上下文协议 | 正文对比实验 |
| `legacy_archive` | `test1-test7_merged` | 历史回顾与方法演进 | 仅背景回顾，不作最终结论 |

当前稳定评估集规模为 1000 篇，主标签工作簿位于：

`Data/Urban Renovation V2.0_cleaned_article_sample_1000_local_labeled_v2_20260407/labels/Urban Renovation V2.0_cleaned_article_sample_1000_local_labeled_v2_20260407.xlsx`

该工作簿当前已闭环的正式标签仅包括“是否属于城市更新研究”；`theme_gold`、`theme_gold_source` 和 `review_status` 字段已经预留，但仍为空，因此主题任务暂不具备正式闭环量化条件。

### 2.3 可复现实验约束

为避免不同轮次结果混用，论文采用如下口径。

1. 三方法主对比矩阵统一取自 `baseline_20260409_finalstable`，因为该轮包含：
   - `local_topic_classifier`
   - `three_stage_hybrid --hybrid-llm-assist off`
   - `three_stage_hybrid --hybrid-llm-assist on`
2. 锁定稳定版结果单独引用 `baseline_20260414_further`，作为最终部署版本，不与研究矩阵直接混作一张主表。
3. 历史集 `test1-test7_merged` 只用于说明方法如何演进，不用于得出当前系统最终性能结论。

## 3. 任务一：城市更新识别实验设计

### 3.1 任务定义

城市更新识别任务是论文主任务，目标是在仅输入题名、摘要及少量元数据的条件下，判断文献是否以城市更新、城市再生、再开发、存量建成环境改造等议题为核心研究对象。该任务的最终输出并不直接依赖单一二分类器，而是由主题级输出 `topic_final` 统一派生得到：

- `topic_final ∈ {U1-U15, N1-N10, Unknown}`
- `urban_flag` 与 `final_label` 由 `topic_final` 派生
- `Unknown` 既是分类不确定状态，也是人工复核池入口

这一设计使主任务不仅能够输出 0/1 标签，还能同时保留主题解释、Unknown 审核和决策来源分析能力。

### 3.2 方法分组

主对比矩阵固定为三种方法。

1. `local_topic_classifier`
2. `three_stage_hybrid --hybrid-llm-assist off`
3. `three_stage_hybrid --hybrid-llm-assist on`

其中：

- `local_topic_classifier` 仅使用本地词项打分与二分类概率，不调用 LLM。
- `three_stage_hybrid --hybrid-llm-assist off` 将规则过滤、局部主题分类器与 BERTopic 辅助信号整合，但 Unknown 恢复不使用 LLM 家族提示。
- `three_stage_hybrid --hybrid-llm-assist on` 在前述基础上，仅对 Unknown 路径收集 LLM 家族提示；该提示不直接改写 `topic_final`，只参与 `unknown_hint_resolution`。

补充实验中保留 `pure_llm_api`，但只作为附录基线，用于估计纯 API 上界和成本，不进入稳定版主表。

### 3.3 评估指标

城市更新识别任务采用以下指标体系。

- 总体指标：Accuracy、Precision、Recall、F1、TP、TN、FP、FN
- 不确定性指标：Predicted Unknown Count、Predicted Unknown Rate
- 路由解释指标：Decision Source Metrics
- 主题分布指标：Topic Distribution

其中 Unknown 采用项目当前正式协议：Unknown 样本计入 Accuracy 的总分母，但不计入 TP/TN/FP/FN。

## 4. 任务一：城市更新识别实验结果

### 4.1 三方法主对比矩阵

表 1 给出 `baseline_20260409_finalstable` 的三方法正式结果。该轮结果来自同一 1000 篇评估集和同一评估脚本，适合用于论文主对比实验。

| 方法 | Accuracy | Precision | Recall | F1 | FP | FN | Predicted Unknown |
| --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| `local_topic_classifier` | 80.5 | 0.945607 | 0.878238 | 0.910678 | 39 | 94 | 62 |
| `three_stage_hybrid + LLM off` | 78.4 | 0.945869 | 0.908345 | 0.926727 | 38 | 67 | 111 |
| `three_stage_hybrid + LLM on` | 83.9 | 0.949266 | 0.913882 | 0.931238 | 38 | 67 | 56 |

从表 1 可以看出，`three_stage_hybrid + LLM on` 在 Precision、Recall、F1 三项核心指标上均优于另外两种方法，并且在 FP 保持不变的情况下显著压缩了 Unknown 留存数量。相较于 `local_topic_classifier`，其 FP 从 39 降为 38，FN 从 94 降为 67；相较于 `LLM off`，其 FP 和 FN 不变，但 Unknown 从 111 下降到 56，从而使整体 Accuracy 从 78.4 提升到 83.9。

这说明三阶段混合策略的主要收益不来自放宽判定阈值，而来自 Unknown 恢复链路的有效启用，即 H1 与 H2 在当前研究矩阵中得到支持。

### 4.2 统计稳健性分析

基于同一 1000 篇输出文件，采用 bootstrap 5000 次重采样计算 Accuracy 和 F1 的 95% 置信区间；同时对三方法进行两两 McNemar 精确检验。结果如下。

| 方法 | Accuracy 95% CI | F1 95% CI |
| --- | --- | --- |
| `local_topic_classifier` | [78.1, 82.9] | [0.895460, 0.925653] |
| `three_stage_hybrid + LLM off` | [75.8, 80.8] | [0.912080, 0.939759] |
| `three_stage_hybrid + LLM on` | [81.5, 86.2] | [0.917663, 0.943937] |

| 比较组 | McNemar `c01` | McNemar `c10` | p 值 |
| --- | ---: | ---: | ---: |
| `local_topic_classifier` vs `hybrid + LLM off` | 28 | 7 | 0.00051 |
| `local_topic_classifier` vs `hybrid + LLM on` | 17 | 51 | 0.000045 |
| `hybrid + LLM off` vs `hybrid + LLM on` | 0 | 55 | < 1e-15 |

其中 `c01` 表示前者正确、后者错误的样本数，`c10` 表示前者错误、后者正确的样本数。三组检验均显示差异具有统计意义，尤其是 `hybrid + LLM on` 相对 `hybrid + LLM off` 没有出现新增退化样本，而是净恢复了 55 个此前落入 Unknown 的案例。

### 4.3 锁定稳定版结果

论文最终系统结论不直接采用表 1 的研究矩阵，而是采用 `baseline_20260414_further` 作为锁定稳定版。其结果如下。

| 运行版本 | Accuracy | Precision | Recall | F1 | FP | FN | Predicted Unknown |
| --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| `three_stage_hybrid + LLM on` (`2026-04-14`) | 85.4 | 0.949934 | 0.914975 | 0.932127 | 38 | 67 | 41 |

与 `baseline_20260409_finalstable` 相比，锁定版在 Precision 和 Recall 小幅提升的同时，将 Unknown 数量从 56 进一步压缩到 41，并满足当前项目设定的全部稳定版验收阈值：

- `Precision >= 0.949`
- `Recall >= 0.914`
- `FP <= 38`
- `Predicted Unknown Count <= 41`
- `llm_used == 0`
- `unknown_hint_resolution` 子集准确率 `== 100%`

因此，论文正文主结论应以 `baseline_20260414_further` 为最终版本，以 `baseline_20260409_finalstable` 为主对比矩阵。

### 4.4 决策来源分解与 Unknown 恢复效果

锁定稳定版的决策来源分解见表 4。

| 决策来源 | 样本数 | Accuracy | Precision | Recall | F1 | Unknown Topic Rate |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| `rule_model_fusion` | 866 | 88.1062 | 0.946915 | 0.909091 | 0.927618 | 0 |
| `stage1_rule` | 17 | 100.0000 | 0.000000 | 0.000000 | 0.000000 | 0 |
| `stage2_classifier` | 5 | 60.0000 | 0.750000 | 0.750000 | 0.750000 | 0 |
| `unknown_hint_resolution` | 71 | 100.0000 | 1.000000 | 1.000000 | 1.000000 | 0 |
| `unknown_review` | 41 | 0.0000 | 0.000000 | 0.000000 | 0.000000 | 1 |

其中最关键的现象是 `unknown_hint_resolution` 路径在 71 个样本上实现了 100% 准确率，且这一路径并不改变 `llm_used == 0` 的最终发布约束，因为 LLM 只提供家族提示，不直接覆写 `topic_final`。这表明 Unknown 恢复链路不是“更激进地把样本判正”，而是在规则、局部主题、BERTopic 和家族提示一致时才执行受限回收。

若与 `baseline_20260409_finalstable` 对比，`unknown_hint_resolution` 的有效回收样本数从 56 个提升到 71 个，而人工复核池从 56 个进一步下降到 41 个，说明后续规则和主题边界修订主要改善的是高精度恢复能力，而非简单扩大正类覆盖。

### 4.5 误差分析

基于 `baseline_20260414_further` 的 `Urban Error Analysis`，FP 与 FN 的主要来源如下。

| 错误类型 | 主要类别 | 数量 |
| --- | --- | ---: |
| FN | `governance_program_law_under_renewal` | 25 |
| FN | `method_design_evaluation_under_renewal` | 25 |
| FN | `health_social_effects_under_regeneration` | 12 |
| FP | `health_social_effects_under_regeneration` | 11 |
| FP | `governance_program_law_under_renewal` | 9 |
| FP | `method_design_evaluation_under_renewal` | 7 |

这些误差表明系统当前的主要困难不在于显性城市更新项目词汇，而在于以下三类边界样本。

1. 以治理、政策、实施机制为中心，但城市更新对象表达较弱的研究。
2. 以评估、方法、设计、建模为中心，但仍然以更新过程为背景或作用对象的研究。
3. 讨论绅士化、健康、公平、社区影响等后果议题，但更新干预与一般城市变迁之间边界不够清晰的研究。

代表性 FP 案例如下：

- `EVERYDAY LIFE, SPATIALITY AND INNER-CITY DISINVESTMENT IN A UNITED-STATES CITY`
- `A model of tax increment financing adoption incentives`
- `Central-city and suburban migration patterns: Is a turnaround on the horizon?`

代表性 FN 案例如下：

- `Urban policy processes and the politics of urban regeneration`
- `THE URBAN ENTERPRISE ZONE`
- `THE ROLE OF HOUSING POLICY IN THE TRANSFORMATION PROCESS OF CENTRAL-EAST EUROPEAN CITIES`

因此，后续误差修正不应继续通过扩大正例词表完成，而应重点补强“治理/机制/评估/后果”四类边界样本的判别规则与主题阈值。

## 5. 任务二：空间属性抽取实验设计

### 5.1 当前状态

仓库当前已经具备空间抽取执行链路和输出文件，例如：

- `scripts/main_py313.py` 的 `spatial` 模式
- `src/strategies/spatial.py`
- 示例输出：`Data/Urban Renovation V2.0_cleaned_article_sample_1000_local_labeled_v2_20260407/output/spatial_few_20260415_174044.xlsx`

现有输出字段包括：

- `空间研究/非空间研究`
- `空间等级`
- `具体空间描述`
- `Reasoning`
- `Confidence`

这说明空间任务在工程上已经可运行，但当前主评估集尚未提供闭环真值，因此论文不能直接把该任务写成“已完成量化评测”的主结论。

### 5.2 空间任务实验方案

空间任务采用“先补标、后闭环”的实施路径。

1. 从当前 1000 篇稳定评估集抽取 300 篇文献，构建双人独立标注子集。
2. 标注字段包括：
   - `Spatial Study`
   - `Spatial Level`
   - `Spatial Desc`
3. 标注完成后，对 `zero / one / few / cot` 四种提示策略进行统一评测。
4. 评测指标为：
   - `Spatial Study`：Accuracy、Precision、Recall、F1
   - `Spatial Level`：层级匹配准确率
   - `Spatial Desc`：Jaccard ≥ 0.6 的模糊匹配准确率

为使这一方案可直接执行，项目新增脚本 `scripts/generate_spatial_annotation_subset.py`，默认从稳定评估集生成 300 篇空间双人标注子集，并可选合并现有空间预测结果作为辅助分层和预标注参考。

### 5.3 论文写法

在当前版本论文中，空间任务建议采用如下表述。

- 方法层面：空间抽取模块、字段定义、提示策略和评估指标已经固定。
- 实施层面：已生成正式补标方案与双人标注模板，下一阶段完成 300 篇闭环标注后进入量化评测。
- 结果层面：当前只能报告结构化输出样例与实验协议，不能将空间任务写成已有最终准确率。

主题任务当前已经形成 25 类主题体系：

- 城市更新主题：`U1-U15`
- 非城市更新主题：`N1-N10`

### 6.3 正式闭环方案

主题任务的正式量化阶段按以下步骤执行。

1. 在当前 1000 篇主评估集上补齐 `theme_gold`。
2. 对两条输出链路同时评测：
   - `local topic`
   - `hybrid final topic`
3. 报告以下指标：
   - `Theme Metrics`
   - `Theme Confusion`
   - `U-N Family Metrics`
   - `Unknown Rate`
4. 与主任务共享同一 comparability 和 manifest 协议，确保不同运行之间可比。

因此，论文当前版本应采用分阶段写法：主题体系与阶段性分布已完成，主题量化闭环待补标后完成。

## 7. 长上下文稳定性与研究矩阵协议

长上下文实验不属于稳定版门槛，而属于 `research_matrix`。其原因在于跨论文共享上下文可能引入输入顺序敏感性，进而破坏稳定版可复现性。

长上下文实验的正式协议固定为三种顺序。

1. `canonical_title_order`
2. `shuffle_seed_20260415_a`
3. `shuffle_seed_20260415_b`

论文在长上下文部分只报告三组顺序的聚合均值，不引用单次运行。判断标准如下。

- 若 `max_delta_accuracy > 1.5`
- 或 `max_delta_f1 > 0.015`

则认定该方法为 `order_sensitive`，只能保留为研究矩阵结果，不能进入稳定版主结论。

当前仓库中尚未形成满足该协议的正式 `Long Context Stability` 汇总工件，因此论文本版只写协议、判定阈值与实验设计，不写最终结论数值。

## 8. 论文实验评估写作模板

为保证正文结构清晰，实验章节建议按以下顺序写作。

### 8.1 实验设计

本研究构建了一个面向城市更新文献识别的三任务实验框架，包括城市更新识别、空间属性抽取与细粒度主题识别。三项任务共享统一运行入口和离线评估接口。其中，城市更新识别已具备 1000 篇稳定评估集和锁定稳定版结果；空间属性抽取与细粒度主题识别已具备方法框架与结构化输出能力，但正式量化闭环仍依赖新增人工标注。

### 8.2 实验评估

在城市更新识别任务上，研究矩阵结果表明 `three_stage_hybrid + LLM assist on` 在 Precision、Recall 与 F1 上整体优于 `local_topic_classifier` 与 `three_stage_hybrid + LLM assist off`。锁定稳定版 `baseline_20260414_further` 进一步达到 `Accuracy=85.4`、`Precision=0.949934`、`Recall=0.914975`、`F1=0.932127`，同时将 Unknown 样本压缩至 41 个，满足当前工程验收阈值。统计分析显示，该结果相对对比基线具有稳定优势。

### 8.3 误差分析

主任务错误主要集中在治理/政策、方法/评估和社会后果三类边界样本上，说明当前系统已经较好覆盖显性更新项目类文献，但在“更新作为研究对象”与“更新作为背景语境”之间的细微边界上仍存在识别难点。Unknown 恢复链路在锁定稳定版中以 100% 的子集准确率回收了 71 个样本，是当前性能提升的关键来源。

### 8.4 局限性与展望

当前论文的主要局限在于空间属性与主题识别尚未完成闭环标注，因此无法像主任务一样报告最终准确率。下一阶段将首先完成 300 篇空间双人标注子集和 1000 篇 `theme_gold` 补标，随后统一进入正式量化评测。

## 9. 结论

从当前仓库已验证工件出发，论文实验部分可以得出如下结论。

1. 城市更新识别已形成稳定、可复现且满足工程阈值的主结果，锁定版本为 `baseline_20260414_further`。
2. 三阶段混合策略的关键增益来自 Unknown 恢复，而非放松分类边界。
3. 空间属性抽取与细粒度主题识别已经完成方法设计和运行链路搭建，但其正式量化闭环仍需新增人工标注。
4. 长上下文策略应严格留在研究矩阵中，以顺序稳定性约束其论文引用边界。

据此，论文正文可以采用“三任务并列、分阶段闭环”的叙述方式：主任务报告正式量化结果，空间与主题任务报告方法框架、阶段性分析与下一阶段闭环方案。
