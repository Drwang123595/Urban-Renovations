# 实验操作说明

## 实验目标
对论文标题与摘要进行结构化抽取，输出四个字段：
1. 是否属于城市更新研究（1/0）
2. 空间研究/非空间研究（1/0）
3. 空间等级（Country / Province(State) / City / Urban agglomeration / Not mentioned）
4. 具体空间描述（Country-Province(State)-City-more specific unit 或 Not mentioned）

## 三种策略
### 策略1：一次性提问 (single)
- 单轮对话输出 4 列，TAB 分隔  
- 适合快速基线比较  
- **上下文**：每篇论文一个独立对话（Session），互不影响。

### 策略2：分步提问 (stepwise)
- 同一对话中分 3 步输出  
  - 第一步：城市更新（1/0）  
  - 第二步：空间/非空间（1/0）  
  - 第三步：空间等级 + 空间描述（TAB 分隔）  
- **上下文**：每篇论文一个独立对话（Session），论文内部保留三步历史，但**不跨论文记忆**。

### 策略3：长上下文分步提问 (stepwise_long)
- 与策略2相同，分3步输出。
- **上下文**：**跨论文保留历史记忆**。模型会记得之前处理过的论文，直到 Token 达到上限（128k）才会重置并开启新 Session。
- 目的：观察长历史记忆是否对批量提取的一致性有提升。

## 环境准备
在项目根目录创建 .env 文件，配置 API Key：
```
DEEPSEEK_API_KEY=你的key
```

## 运行方式
### 方式一：统一入口（运行后手动选择策略）
```
python main.py --shot zero
python main.py --shot one
python main.py --shot few
```
运行后按提示选择：
- 1：single（单次输出）
- 2：stepwise（分步输出 - 独立Session）
- 3：stepwise_long（分步输出 - 长上下文）

### 方式二：明确指定策略
```
python main.py --shot few --strategy single
python main.py --shot few --strategy stepwise
python main.py --shot few --strategy stepwise_long
```

### 方式三：分步专用脚本
```
python stepwise_experiment.py --shot zero
python stepwise_experiment.py --shot one
python stepwise_experiment.py --shot few
```

## 输入与输出
输入文件默认：  
```
Data/test1.xlsx
```

输出文件默认：  
```
data/output/result_<shot>_<timestamp>.xlsx
```

输出写入列名：
- 是否属于城市更新研究
- 空间研究/非空间研究
- 空间等级
- 具体空间描述

## 常见问题
### 401 Unauthorized
- 检查 .env 是否正确设置
- 确认 Key 有权限访问 https://api.deepseek.com/v1/chat/completions

### 429 Too Many Requests
- 免费额度耗尽或并发过高，请尝试增加 `config.py` 中的重试次数或等待时间。

### 上下文超长警告
- 如果运行 `stepwise_long` 策略，看到控制台输出 `[WARNING] Context length approaching limit!`，说明当前 Session 即将达到 Token 上限，程序会自动开启新 Session，属于正常现象。
