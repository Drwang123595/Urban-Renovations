# 提示词策略长期维护与迁移说明

## 1. 原则
- `src/templates/strategy_registry.yaml` 是唯一可执行事实源。
- 所有运行时提示词必须位于 `src/templates/<theme>/`。
- 任何提示词变更必须通过 `scripts/manage_prompt_strategies.py` 完成，不允许手工直接改注册表后提交。
- 每次变更都必须带上 `version`、`owner`、`change_summary`。
- 新策略默认用 `candidate` 生命周期进入实验，只有经过验证后才能 `promote` 为 `stable`。

## 2. 生命周期
- `candidate`：允许实验，但运行主脚本时必须显式加 `--allow-candidate`。
- `stable`：允许正式实验和常规运行。
- `deprecated`：不允许运行，只保留审计和回溯用途。

## 3. 运行前检查
- `scripts/main.py` 会在启动时解析两个任务的最终策略。
- 默认只允许 `stable` 策略；`candidate` 必须通过 `--allow-candidate` 明确启用。
- 每次运行都会在预测文件旁生成 `<output>.prompt_manifest.json`，记录策略版本、模板哈希、运行模型参数和输入文件。

## 4. 管理命令
```bash
python scripts/manage_prompt_strategies.py list --enabled-only
python scripts/manage_prompt_strategies.py get few --theme urban_renewal
python scripts/manage_prompt_strategies.py add reflection_v2 --theme urban_renewal --template-file reflection_v2.yaml --alias self_check_v2 --description "Reflection template" --version 0.1.0 --owner prompt_owner --change-summary "Add candidate reflection prompt" --system-prompt "system text"
python scripts/manage_prompt_strategies.py update few --theme urban_renewal --owner prompt_owner --change-summary "Refresh few-shot examples" --version 2.2.0
python scripts/manage_prompt_strategies.py promote few --theme urban_renewal --owner prompt_owner --change-summary "Promote tested candidate to stable"
python scripts/manage_prompt_strategies.py deprecate reflection --theme urban_renewal --owner prompt_owner --change-summary "Retire legacy reflection prompt"
python scripts/manage_prompt_strategies.py delete reflection --theme urban_renewal --archive-dir src/templates/_archived
python scripts/manage_prompt_strategies.py check
```

## 5. 推荐工作流
1. 用 `add` 创建候选策略，默认保留为 `candidate`。
2. 通过 `scripts/main.py --allow-candidate` 运行实验，生成预测文件和 prompt manifest。
3. 用 `scripts/evaluate.py` 对比实验结果；比较结果必须读取 manifest，不能只按文件名判断。
4. 达到预期后用 `promote` 提升为 `stable`。
5. 废弃旧策略时用 `deprecate`，确认不再需要后再 `delete` 或归档。

## 6. 回滚
- 快速回滚：把异常策略 `deprecate`，切回上一版 `stable`。
- 模板回滚：从 `src/templates/_archived/` 恢复模板文件，再通过 `update` 恢复注册表映射。
- 回滚后必须重新跑 `python scripts/manage_prompt_strategies.py check` 和对应测试。
